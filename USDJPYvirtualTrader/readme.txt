基本的な事項
証拠金は円建て
初期証拠金は100万円
最大注文件数は5まで
時刻表示はUTC
成行注文・成行決済のみ、指値注文はアプリケーションを閉じている間価格を取得することが出来ないので不可。
ウィンドウを閉じると全てのポジションを決済する。
レバレッジ = 100倍固定
ロスカットレート = 30%
bid: 最も高い買い板の値段 　=　今売る事の出来る値段 
ask: 最も安い売り板の値段	=　今買う事の出来る値段
spread: bidとaskの差
price history: 為替取引(FX)では相対取引が基本であり、板が存在しないのでbidとaskの中間値を取っている
lot: ここでは1lot = 10万ドル = 約1000万円(USDJPY = 100.000で計算) 
Balance: 証拠金 
Equity: 証拠金 + 未実現利益(損失)
Margin: 必要証拠金(ポジションを持つのに必要とされる証拠金額)
ここでは必要証拠金が証拠金以上になるようなポジションを持つことは出来ない。
Margin level: 証拠金維持率
ロスカット: 証拠金以上の損失を防ぐために用いられる強制決済の事で、証拠金維持率がロスカットレートを下回った時に執行される。
ここでは30%を下回った場合、強制的に全てのポジション(建玉)を反対売買により解消する。
ここでは、注文を出す際に証拠金維持率が100%以下になるようなポジションを建てることは出来ない。
CLOSE: 成行決済のボタン。このボタンを押すとポジションを反対売買により閉じることが出来る。

例1:　証拠金1,000,000円で1 lot = 10万ドルを105.000円で売る(空売り(ショート))するとき、
　    必要証拠金は105.000 * 100,000 / 100 = 105,000円
　    ポジションを建てた瞬間の証拠金維持率は Equity / Margin * 100より、1,000,000 / 105,000 * 100 = 952.31 %

例2:　例1のポジションを100.000円で反対売買により決済したとき、
　　  利益は105.000 - 100.000 * 100,000 = +500,000円

例3:　例1のポジションを115.000円で損切りしたとき、
　　　損失は105.000 - 115.000 * 100,000 = -1,000,000円

つまり、10万通貨(ここでは10万ドル)の注文で1円幅とれば10万円の利益、逆行すれば10万円の損失となる。

注意:ここではoanda.comよりrest apiを使ってデータを取得している。最終課題品評会ではapiの制限(120requests / 1sec)に違反しないよう、データの取得を1秒に1回程度に制限する。このため、ボタンを押してから画面に反映されるまで最大1秒間のラグが発生する
過去のローソク足をapiから取得しているが、かなりデータの欠損が見受けられるので実際のチャートとは異なる場合がある
また、oandaのapiは口座番号、apikeyが必要になるが、品評会が終わった後にapikeyを破棄するため品評会後はこのプログラムを動かすことは出来ない
